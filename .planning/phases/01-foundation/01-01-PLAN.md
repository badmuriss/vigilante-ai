<task type="auto" tdd="true">
  <name>Task 2: Fix stop/start crash, thread safety, and FPS throttling</name>
  <files>backend/app/stream.py, backend/app/camera.py, backend/app/alerts.py</files>
  <behavior>
    - All tests from Task 1 pass (GREEN phase)
    - StreamProcessor uses threading.Event instead of bool _running
    - StreamProcessor has _epoch counter incremented on each start()
    - generate_mjpeg checks both _stop_event and epoch match
    - stop() clears _current_jpeg to b""
    - _process_loop has FPS throttling at TARGET_FPS=25
    - AlertManager.record_frame and _is_on_cooldown use _lock
    - StreamProcessor._fps and _start_time reads are lock-protected
    - CameraManager uses threading.Event for _running
  </behavior>
  <action>
    **stream.py changes:**
    1. Replace `self._running = False` with `self._stop_event = threading.Event()`. Add `self._epoch: int = 0`.
    2. `is_running` property: `return not self._stop_event.is_set()`
    3. `start()`: Call `self._stop_event.clear()`, increment `self._epoch`, then start camera and thread as before.
    4. `stop()`: Call `self._stop_event.set()`, join thread, stop camera. Clear `self._current_jpeg = b""`. Reset `_fps = 0.0`, `_start_time = 0.0`.
    5. `generate_mjpeg()`: Capture `epoch = self._epoch` at start. Loop condition: `while not self._stop_event.is_set() and epoch == self._epoch`. Replace `time.sleep(0.03)` with `self._stop_event.wait(0.03)` (returns immediately on stop).
    6. `_process_loop()`: Add FPS throttling -- `TARGET_FPS = 25`, `FRAME_INTERVAL = 1.0 / TARGET_FPS`. At end of each iteration, compute `sleep_time = FRAME_INTERVAL - elapsed` and `self._stop_event.wait(sleep_time)` if positive. Loop condition: `while not self._stop_event.is_set()`.
    7. Protect `_fps` and `_start_time` reads with `self._lock` in properties `fps` and `uptime`.

    **camera.py changes:**
    1. Replace `self._running = False` with `self._stop_event = threading.Event()`.
    2. `is_running` property: `return not self._stop_event.is_set()`
    3. `start()`: `self._stop_event.clear()` before starting thread.
    4. `stop()`: `self._stop_event.set()`, join thread, release capture.
    5. `_capture_loop()`: Loop on `while not self._stop_event.is_set() and self._capture is not None`.

    **alerts.py changes:**
    1. Wrap `record_frame` body in `with self._lock:` to protect `_total_frames` and `_compliant_frames` increments.
    2. Wrap `_is_on_cooldown` reads of `self._cooldowns` in `with self._lock:`.
    3. Wrap `get_stats` reads of `self._total_frames`, `self._compliant_frames` in `with self._lock:`.

    Do NOT change the detection logic (VIOLATION_CLASSES, etc.) -- that is Plan 02's scope. Keep existing import of `VIOLATION_CLASSES` in stream.py for now.
  </action>
