    - test_stream.py::test_alert_uses_portuguese_label: Alerts generated by the process loop use EPI alert labels (e.g., "Capacete ausente") not raw class names.
    - test_stream.py::test_missing_epi_alert: When at least one EPI is detected but another active EPI is absent, an alert is generated for the missing EPI.
    - test_stream.py::test_no_alert_empty_frame: When zero EPIs are detected in a frame (no person proxy), no "missing EPI" alerts are generated.
  </behavior>
  <action>
    **stream.py changes:**
    1. Add `self._active_epis: set[str] = set()` and `self._epi_lock = threading.Lock()` to `__init__`.
    2. Add property `active_epis` (getter with lock) and setter `set_active_epis(epis: set[str])` (setter with lock).
    3. In `_process_loop`, after `detections = self._detector.detect(frame)`:
       a. Copy active EPIs under lock: `with self._epi_lock: active = self._active_epis.copy()`
       b. Filter detections: `filtered = [d for d in detections if d.class_name in active] if active else []`
       c. Annotate frame with `filtered` only (not all detections).
       d. Alert logic (replaces old VIOLATION_CLASSES logic):
          - If `filtered` is non-empty (person proxy: at least one active EPI detected), check which active EPIs are MISSING from detections.
          - `detected_keys = {d.class_name for d in filtered}`
          - `missing_keys = active - detected_keys`
          - For each missing key, call `self._alert_manager.add_alert(EPI_ALERT_LABELS[key], confidence=0.0, frame=frame)` where confidence=0.0 indicates absence-based alert.
       e. `is_compliant = len(missing_keys) == 0 if filtered else True` (no person = compliant by default).
       f. `self._alert_manager.record_frame(compliant=is_compliant)`
    4. Remove old `from app.detector import VIOLATION_CLASSES` import and the violation filtering logic.
    5. Import `EPI_ALERT_LABELS` from detector.py.
    6. In `stop()`, also call `self._alert_manager.clear_alerts()` per CONTEXT.md decision: "Stopping monitoring resets all alerts."

    **alerts.py changes:**
    1. Add `reset_session()` method that clears alerts, cooldowns, resets counters, and resets `_session_start`. Called by `stream_processor.stop()`.

    1. Configure mock_detector to return specific detections (e.g., capacete + luvas).
